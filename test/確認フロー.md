
# 確認フロー

全体像が机上どおり動くか、以下の手順で確認する。
テストコードは [`test/`](test/) ディレクトリに置く。

```
STEP 1  ブローカ起動（管理PC）
STEP 2  Unity 接続テスト（subscribe 確認）
STEP 3  MQTT クライアントで手動 publish（ブローカ疎通確認）
STEP 4  ESP32 書き込み・電源投入（hb 受信確認）
STEP 5  ESP32 入力 → Unity 受信確認（evt）
STEP 6  Unity → ESP32 送信確認（cmd）
```

## STEP 1: Mosquitto（ブローカ）を管理PC で起動→ok

```powershell
# インストール済みの場合
mosquitto -v
# -v で全メッセージをターミナルに表示（デバッグ用）
```

> インストール: https://mosquitto.org/download/  
> `mosquitto.conf` に `listener 1883` と `allow_anonymous true` を追加する。
> https://rikoubou.hatenablog.com/entry/2024/03/25/165952　

## STEP 2: Unity 側テストスクリプトを起動→ok
unity 6.0(6000.0.68f1)を使用。

1. [×]Unity Hub からプロジェクト `unity_test/` を開く（このリポジトリ直下）
2. [×]MQTT ライブラリ（M2Mqtt）を Unity に導入する（どちらか）
   - GitHub: M2MqttUnity（`M2MqttUnity` をプロジェクトにインポート）
   - または: NuGetForUnity 経由で `M2Mqtt` を入れる
3. [×]テストスクリプトを Unity の `Assets/` 配下に置く
   - `unity_test/Assets/Scripts/` を作成
   - `test/unity/MqttConnectionTest.cs` を `unity_test/Assets/Scripts/MqttConnectionTest.cs` としてコピー
4.[×]`unity_test/Assets/Scenes/SampleScene.unity` を開く（未保存なら保存）
5.[×]Hierarchy で空の GameObject を作成（名前例: `MqttConnectionTest`）
6.[×]Inspector で `MqttConnectionTest (Script)` コンポーネントを追加（Add Component から検索）
7.[×]Inspector の設定
   - `brokerAddress`
     - ブローカ（Mosquitto）と同じPCで Unity を動かすなら `127.0.0.1`
     - 別PCなら「ブローカPCの IPv4 アドレス」（例: `192.168.0.10`）
   - `brokerPort` は通常 `1883` のまま
8.[×]Play → Console で以下が出れば接続成功
   - `[MQTT] Connected to <IP>:1883`
   - `[MQTT] Subscribed: yugo/#`

うまくいかないとき（よくある原因）
- Unity を別PCで動かしているのに、ブローカが「local only mode」になっている（設定ファイルで `listener 1883` を有効にしてブローカを再起動）
- Windows ファイアウォールで 1883/TCP がブロックされている（ブローカPC側で受信許可）
- `brokerAddress` が間違い（`127.0.0.1` は「そのPC自身」なので、別PCのブローカにはつながらない）

## STEP 3: MQTT クライアントで手動 publish（ブローカ疎通確認）→ok

```powershell
# mosquitto_pub で手動送信（mosquitto インストールに含まれる）
mosquitto_pub -h 127.0.0.1 -t "yugo/stage1/nodeA/evt" `
              -m '{"t":"touch","id":"fish","v":1,"ts":0}'
```

Unity の Console に `[EVT] yugo/stage1/nodeA/evt ...` が表示されれば **ブローカ ↔ Unity 間 OK**。

## STEP 4: ESP32 書き込み・hb 確認→ok

1. [`test/esp32/mqtt_connection_test/mqtt_connection_test.ino`](test/esp32/mqtt_connection_test/mqtt_connection_test.ino) の `WIFI_SSID` / `WIFI_PASS` / `MQTT_BROKER` を書き換える
2. Arduino IDE または PlatformIO で ESP32 に書き込む
3. シリアルモニタ（115200）で `[WiFi] Connected` / `[MQTT] connected` を確認
4. Unity Console に `[HB ]` ログが 1秒ごとに流れれば **ESP32 ↔ ブローカ ↔ Unity 間 OK**

## STEP 5: ESP32 入力 → Unity 受信（evt）→ok

- D15 ピンを GND に触れる（またはボタン押下）
- ESP32 シリアルモニタに `[EVT] ->` ログが出る
- Unity Console に `[EVT] yugo/stage1/nodeA/evt {"t":"touch","id":"fish"...}` が表示される

## STEP 6: Unity → ESP32 送信（cmd）→ok

- Unity が Play 中に **Space キー** を押す
- Unity Console に `[CMD] -> yugo/stage1/nodeA/cmd {"t":"led","id":"tree","v":1}` が出る
- ESP32 の `LED_BUILTIN` が点灯し、シリアルモニタに `[CMD] t=led id=tree v=1` が出る
